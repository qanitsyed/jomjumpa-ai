// PROMPT LIBRARY

Feature prompts that just work.

Copy any of these, paste into your AI chat, watch the magic. Each is designed to add one specific capability without breaking the rest.

UX Add dark mode
Add a dark mode toggle in the top-right corner. Use CSS variables so colours switch cleanly. Remember the user's choice with localStorage. Keep the rest of the app unchanged.
DATA Save data between visits
Use localStorage to save all user entries so they persist when I close the browser and come back. Load them automatically on page open.
VISUAL Add a chart
Add a simple bar or line chart showing the data over time. Use inline SVG or Canvas — no external chart libraries. Keep it clean and minimal.
POLISH Make it look more polished
Improve the visual design. Use a clean modern aesthetic with proper spacing, a refined colour palette (max 3 colours), and good typography. Mobile-friendly. Same single HTML file.
FUNCTION Add export / share
Add a button to download my results as a PNG image, or copy a shareable summary to my clipboard. Use built-in browser APIs only.
I18N Add a language toggle
Add a toggle in the top-right that switches the interface between English and Bahasa Melayu. Translate all labels and buttons cleanly.
// COMMON GOTCHAS

When things go sideways.

001 AI broke my working app ▸

Don't panic. Say to your AI: Revert the last change — go back to the previous working version.

If that doesn't work, scroll up in your chat to the last good code block and copy that one instead. Your old code is still there in the chat history.

002 My Netlify URL changes every deploy ▸

You're getting fresh URLs because each drop creates a new "site". To fix:

Create a free Netlify account → on your most recent deploy, click Site settings → Change site name → pick something memorable like my-tracker-app. Now redeploying to the same site keeps the URL.

003 App is getting too complex / slow ▸

Single-file HTML hits a ceiling around 1000–1500 lines of code. If you're past that, your AI may struggle to keep all parts working together.

Two options: (a) Trim features — keep only what you really use. (b) Move to a real builder — see Track 4.

004 AI added a feature but the design feels off ▸

The design drift is real. Two prompts that fix it:

Restyle the new feature to match the existing design language.

Polish the whole UI — keep all functionality identical but make the visual design cohesive and modern.

005 I want to add multiple features at once ▸

Resist the urge. Asking for 3 features in one prompt has a much higher break rate.

Add one. Test it. Save the working version. Then add the next.

// FULL WALKTHROUGH

The slow version, if you need it.

STEP 01
Go back to your AI chat
Use the same conversation where you built the app. Don't start a new one — your AI will lose context.
STEP 02
Describe one change
Use a prompt from the library above, or describe your own change in one sentence. Specificity wins: "add a green button labelled 'Save'" beats "make it better".
STEP 03
Test in the preview
Click around in the AI's preview. Did the new feature work? Did anything else break? If broken, ask the AI to fix the specific issue.
STEP 04
Download the new code
Just like Track 1 — use the Download button/icon (web: hover code block; Claude mobile: ⋯ on artifact; ChatGPT mobile: download icon). It saves to your Downloads folder.
STEP 05
Check the filename
Your downloaded file should be named index.html. If it has a different name, rename it. No need to open it — the new code is already inside.

If download didn't work: copy the code from your AI instead, and use the paste route at pages.edgeone.ai/drop in the next step.
STEP 06
Redeploy
Upload route (try first): Go to Netlify Drop and upload your file again. If you've claimed a Netlify site name, go to that site's Deploys tab and drop there to keep the same URL.

Paste route (if upload won't work / on phone): Open pages.edgeone.ai/drop → Paste Code → paste → submit.
STEP 07
Refresh and admire
Reload your URL. Your changes are live to the entire internet. Repeat from Step 02 with the next feature.
