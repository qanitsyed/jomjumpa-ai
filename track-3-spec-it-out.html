TRACK 03 · SPEC IT OUT

Brief the next agent.

One markdown file at the root of your repo. Every AI tool you bring in tomorrow already knows what it's looking at — without you re-explaining anything.

The Prompt Simplifier is the prompt's spec. AGENTS.md is the repo's spec. Both do the same job at different scales.

// THE PROBLEM

Your AI keeps forgetting.

Every new chat starts at zero. You re-explain the project. You repeat your conventions. You remind it not to use React, again. Multiply that by every session, every new agent, every new tool you try.

The fix is almost embarrassingly simple. Drop a single markdown file at the root of your repo. Every modern AI dev tool will read it automatically before the first message. The repo briefs the agent, so you don't have to.
// THE FIX

One file. One read.

The file is called AGENTS.md. It lives at the root of your repo — the same repo you set up in Track 2, sitting right next to your index.html. It's just markdown. There's no framework to install, no syntax to learn, no setup.

When you start a new AI session against this repo, the assistant reads AGENTS.md first. Your conventions, your gotchas, your file map are loaded into the conversation before you even say hello.

// STRUCTURE

Five sections that earn their keep.

A good AGENTS.md is short. Three pages, max. If it's longer, the agent will skim and miss the important parts. Aim for this shape:

1
Project overview
One paragraph. What this is, who it's for, what makes it different from the obvious assumption.
2
How to run it
The exact commands to install, run, and test. Copy-pasteable. The agent will use these without asking.
3
Conventions
How you like things written. Style, naming, structure, any taste calls the agent should match.
4
Don't do this
The anti-patterns. The libraries you've ruled out. The rabbit holes you've already explored. This section saves the most time.
5
File map
Where things live. Three to ten lines. Lets the agent navigate without grepping.
// EXAMPLE

What a real one looks like.

This is a trimmed version of the AGENTS.md for the very site you're reading. Notice how short it is — and how much it would save the next agent from guessing.

AGENTS.md
# Jom Jumpa #AI Workshop site for an AI-assisted coding meetup in Singapore. Pure static HTML, one file per page. No build step. Hosted on Netlify. ## How to run python3 -m http.server 8000 # then visit http://localhost:8000 ## Conventions - One self-contained HTML file per page. All CSS and JS inline. - No external libraries, no build tools, no frameworks. - Fraunces (serif) for headings, Manrope for body, JetBrains Mono for code. - Track colours come from the design tokens at the top of index.html. ## Don't do this - Don't add React, Tailwind, or any framework. - Don't introduce a build step. - Don't create new pages without updating the index card. ## File map index.html — landing page, track cards, built-tonight feed track-1-get-it-live.html — the beginner flow track-2-dev-mode.html — GitHub + Netlify auto-deploy track-4-agent-mode.html — Codespaces + Supabase + Claude Code supabase/functions/ — Tally webhook Edge Function
Notice what's missing. No architecture diagrams. No history. No story. Just the operational stuff — what the agent needs to do useful work in the next minute. Save the philosophy for the README.
// HANDS-ON

Write yours in ten minutes.

Use the AI to draft the first version — it knows your code better than you remember it. Then edit by hand. The hand-editing is where the real value lands.

1
Go back to the AI chat where you built your app

Open the same Claude or ChatGPT conversation you used in Track 1 (or wherever your app's code lives). The AI already has full context for your project.

No code in chat? Paste your current index.html into a fresh message first.

2
Ask for the first draft

Copy and paste this prompt — it's tuned to give you a focused, opinionated draft instead of a generic template:

Read my project and write me an AGENTS.md file for it. Keep it under 80 lines. Include: a one-paragraph project overview, how to run it locally, conventions you can infer from the code, three or four anti-patterns to avoid based on the style I'm using, and a file map. Be specific. Don't include sections that don't apply.
3
Edit by hand — especially "Don't do this"

The AI guesses your conventions from the code. It will be 80% right. Spend five minutes on the "Don't do this" section — that's where your real taste lives and where the AI can't help you. Add the anti-patterns you've personally learned the hard way tonight.

4
Add it to your GitHub repo

Open your repo on github.com (the one from Track 2). The file goes at the root of the repo — same level as your index.html, not inside any folder.

Using the GitHub web UI:

  1. Click the green Add file button near the top-right of the file list.
  2. Pick Create new file.
  3. In the filename box, type AGENTS.md — capitalisation matters.
  4. Paste your content into the editor below.
  5. Scroll down. Type a short commit message (e.g. "Add AGENTS.md"). Click Commit new file.

That's it. The file lives in your repo. Every future AI session that reads this project will pick it up automatically.

5
See it earn its keep

Open a brand new AI chat — not the one you've been using. Paste your AGENTS.md as the first message, then ask something vague like "add a contact form to my site" or "clean up the homepage."

Notice how the AI matches your conventions without being told. That's the file working.

In Track 4, you'll meet tools that read this file automatically, no pasting required — the same magic, hands-free.

// THE DIFFERENCE

Same prompt. Two responses.

Asked "add a contact form to the homepage" in two fresh sessions — one with AGENTS.md, one without. The reply you'd actually want is the right one.

WITHOUT AGENTS.MD

"I'll add a React component using Formik for the form. Should I install Tailwind for the styling? I'll set up a Vite build so we can run it properly."

WITH AGENTS.MD

"Adding an inline form to index.html — pure HTML + CSS, no libraries, matching the existing Fraunces/Manrope type scale. Submissions go to your existing Tally webhook."

// GOING DEEPER

When one file isn't enough.

AGENTS.md is the floor, not the ceiling. Once you've felt the value, here are the next layers — each adds on top, none replaces what you've already done.

→
Other AI tools read this file too. Every modern AI coding tool has a preferred filename, but most also respect AGENTS.md as a fallback. You don't need to know any of these tonight — just file them for later.

AGENTS.md (this is where the name comes from)
→
Skills & slash commands. Save reusable prompts as small files inside your repo — one-line triggers for multi-step workflows ("run the migration check," "draft a release note"). Claude Code calls these "skills"; Cursor calls them "rules"; Copilot calls them "instructions". Same idea, different label.
→
Subagents / personas. Specialised agents inside your repo for narrow jobs (security review, schema design, performance audit). Each gets its own context, tools, and conventions.
→
Spec-driven development. Heavier methodology — specs → plans → tasks → implementation. The leading framework is spec-kit. Worth it for serious projects; overkill for a one-night workshop.
→
Per-folder context. A nested AGENTS.md inside a subdirectory overrides the root one for files within. Useful when one repo holds very different kinds of code.
