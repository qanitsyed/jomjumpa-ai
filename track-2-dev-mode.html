Track 02 — Dev Mode

GitHub repo, continuous deployment, your URL forever. The same workflow a real developer uses, with you doing none of the actual coding.

Heads up: This track adds 4–6 new tools to your workflow. Expect ~20 minutes if everything goes smoothly. The reward: any future change to your code auto-deploys to your live site.
📱 On a phone? Both GitHub and Netlify work fully in mobile browsers — every step below works on a phone. Use the browser's "Desktop site" mode if any page looks cramped.
§ 2.1 · OVERVIEW

What you're actually building

A continuous deployment pipeline. When you push code to GitHub, Netlify detects the change and rebuilds your site automatically.

# The mental model your AI → code code → GitHub repo GitHub repo → Netlify (auto-deploy) Netlify → live URL
§ 2.2 · PREREQUISITES

Before you start

REQUIRED
  • A working index.html from Track 1 (or any single-file app you've built)
  • An email address you can verify quickly
  • ~20 minutes of focus
NOT REQUIRED
  • Knowledge of git command line — we'll use GitHub's web UI
  • A terminal — everything happens in the browser
  • A paid plan — both GitHub and Netlify are free for this
§ 2.3 · PROCEDURE

Step-by-step

Each step expands on the one before. Don't skip — verify each step worked before moving on.

01
Create a GitHub account

Go to github.com/signup. Pick a username you don't mind being public — it'll appear in your repo URLs. Verify your email when prompted.

02
Create a new repository

Top-right corner → + icon → New repository.

Repository name
your-app-name
Visibility
Public
Initialize with README
✓ Yes

Click Create repository.

03
Upload your index.html

On your new repo page → Add file → Upload files.

Drag your index.html file onto the upload area, or click to choose it. Scroll down → Commit changes.

Your file is now in version control. Every future change creates a new "commit" — a snapshot you can always revert to.

04
Sign up for Netlify

Go to app.netlify.com/signup. Sign up with GitHub — this saves you a step later by pre-authorising the connection.

05
Import from GitHub

On Netlify dashboard → Add new site → Import an existing project → Deploy with GitHub.

Authorise Netlify when prompted. Then pick your repo from the list.

06
Configure deployment

Netlify will show a deployment settings page. For a plain HTML site, you can leave everything blank or default:

Branch to deploy
main
Build command
(leave empty)
Publish directory
(leave empty)

Click Deploy site.

07
Claim your URL

Netlify gives you a random URL like fluffy-banana-1234.netlify.app. Change it: Site configuration → Site information → Change site name.

Pick something like my-zakat-app. That's now your permanent URL.

08
Test the auto-deploy

Back to GitHub → open your index.html → click the pencil (edit) icon → make any tiny change (add a space, change a word). Commit.

Wait ~30 seconds. Refresh your Netlify URL. Your change is live. That's continuous deployment.

§ 2.4 · TROUBLESHOOTING

Common errors

ERR_001 GitHub didn't send a verification email ▸

Check spam. Then check the email address you typed. GitHub verification emails come from noreply@github.com.

If still nothing in 5 min, log out and try a different email — the registration may have silently failed.

ERR_002 Netlify deployed but site shows "Page not found" ▸

Your file is probably named wrong. It must be index.html, all lowercase, no extra extension.

Check on GitHub: does your repo show index.html in the file list? If it's Index.html or index.html.txt, rename it.

ERR_003 Netlify can't see my GitHub repo ▸

Authorisation issue. Go to app.netlify.com/user/applications → GitHub → Configure → grant access to the specific repo or "All repositories".

ERR_004 My commit didn't trigger a deploy ▸

Check the Netlify Deploys tab. You should see a new deploy started within seconds of your commit.

If not, you may have committed to a branch other than main. Netlify only auto-deploys the branch configured in Site settings → Build & deploy.

ERR_005 I want to delete this whole thing ▸

Two separate things to delete:

Netlify site: Site configuration → bottom of page → Delete this site.

GitHub repo: Repo Settings → bottom of page → Delete this repository. Type the repo name to confirm.

ERR_006 How do I update my code via my AI now? ▸

Two patterns:

(a) Update via GitHub web UI: Get new code from your AI → on GitHub, open your repo → click index.html → click the pencil (edit) icon → select all, delete, paste your new code → Commit changes. Netlify auto-deploys. Note: the Upload option is only for adding new files — for updating an existing file you must use the pencil editor.

(b) Use a proper editor: Track 4 Path B (Codespaces) gives you a full VS Code environment wired directly to this repo. Edit, commit, push — Netlify auto-deploys. No manual upload at all.
