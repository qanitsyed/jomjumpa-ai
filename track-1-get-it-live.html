SOMETHING NOT WORKING?

Tap whatever sounds like your problem.

Most issues are tiny and have a one-line fix. Find yours below.

🤖
My AI keeps using React or strange imports
▸

This means it ignored the Prompt Simplifier. Send this exact message:

Rewrite this as a single plain HTML file with all CSS and JavaScript inline. No imports, no React, no external libraries. Mobile-friendly.
📥
How do I get the code out — download or copy?
▸

Download is the easy path. Look for a download icon near the code block.

💻 On a laptop (Claude or ChatGPT web): Hover over the code block — a Download button appears. Tap it. The file lands in your Downloads folder.

📱 Claude mobile app: Tap the three dots (⋯) at the top-right of the artifact preview. Pick Download (or Copy, if you prefer).

📱 ChatGPT mobile app: Tap the download icon below the code block. (The mobile ChatGPT app doesn't make Copy easy — Download is your route.)

If download doesn't work for any reason, copy the code and use the paste-route via EdgeOne — see the "I'm on my phone" troubleshoot below.

💾
Where's my file? Do I need to rename it?
▸

Try downloading first. Your file is in your Downloads folder. The AI usually names it index.html already — perfect.

If it's named something else (like document.html or app.html), just rename it to index.html. No need to open it or edit anything inside.

If download doesn't work at all: copy the code from your AI, open pages.edgeone.ai/drop, tap Paste Code, paste, and submit. Same outcome, different route.

📱
I'm on my phone — full mobile flow
▸

You've got two paths. Try downloading first.

Path A — Download & upload (try first):

1. Download the file from your AI (see "How do I get the code out?" above)
2. Open your phone's Files app — look in the Downloads folder. Your file should be there.
3. Open netlify.com/drop in your phone browser → tap to choose the file → upload. Get a URL.

Path B — Copy & paste (if you can't find or upload the file, or you're on iPhone):

1. Copy the code from your AI (Claude mobile: ⋯ → Copy)
2. Open pages.edgeone.ai/drop
3. Tap Paste Code → paste → submit. Get a URL.

⚠️
Netlify says error / nothing loads
▸

The most common cause: your file isn't named exactly index.html. Even Index.html or index.html.txt will break it.

Check the file name in your Downloads folder. Lowercase, ends in .html, no extra extensions. Rename if needed and try uploading again.

If that's right and the page still doesn't load — wait 30 seconds and refresh. Netlify sometimes takes a moment.

PREFER A FULL WALKTHROUGH?

The slow, step-by-step version.

i.
Pick a small idea
Keep it tiny. A calculator, a tracker, a tool you'd actually use. The smaller the idea, the faster the win.
ii.
Open your AI
Go to claude.ai or chat.openai.com. Both work for this.
iii.
Type your prompt
Describe your idea in plain English. Then paste the Prompt Simplifier (the block at the top of this page) on the next line. Send. The AI will reality-check your idea and propose the closest version that runs in one browser tab.
iv.
Check the preview
The AI should show a working preview. Try it. Does it do roughly what you wanted? If not, tell it what to change. Don't overthink — keep iterating.
v.
Download the code file
Look for a Download button or icon near the code block (web: hover over the code; Claude mobile: ⋯ menu on artifact preview; ChatGPT mobile: download icon below code). The file goes to your Downloads folder.
vi.
Check the file is named index.html
Open your Downloads folder. The file is usually already named index.html. If not, rename it. No need to open it or edit anything inside — your code is already there.
vii.
Upload it to Netlify
Go to netlify.com/drop. Drag your file onto the page (or tap to choose it on mobile). Wait 10 seconds. You'll see a live URL.

📱 Phone can't upload the file? Go back, copy the code from your AI, open pages.edgeone.ai/drop, tap Paste Code, paste, and submit.
viii.
Share it
Open your URL. It works on the internet now. Anyone with the link can visit. Paste it into the form to add it to the Wall of Apps. That's the whole thing.

THE LOOP

The iterate-redeploy rhythm.

1
Open your existing AI chat. Tell it what to add — one feature at a time.
2
Check the preview. If something broke, say "undo the last change" or describe exactly what went wrong.
3
Happy with it? Download the new file the same way as before.
4
Redeploy: re-upload to Netlify, or re-paste at pages.edgeone.ai/drop.
5
Tip: on Netlify, claim a site name once — your URL stays the same across deploys.
PROMPT LIBRARY

Feature prompts that just work.

Copy any of these, paste into your AI chat, watch the magic. Each adds one capability without breaking the rest.

UX Add dark mode
Add a dark mode toggle in the top-right. Use CSS variables so colours switch cleanly. Remember the user's choice with localStorage. Keep the rest of the app unchanged.
DATA Save data between visits
Use localStorage to save all user entries so they persist when I close the browser and come back. Load them automatically on page open.
VISUAL Add a chart
Add a simple bar or line chart showing the data over time. Use inline SVG or Canvas — no external chart libraries. Keep it clean and minimal.
POLISH Make it look more polished
Improve the visual design. Use a clean modern aesthetic with proper spacing, a refined colour palette (max 3 colours), and good typography. Mobile-friendly. Same single HTML file.
FUNCTION Add export / share
Add a button to download my results as a PNG image, or copy a shareable summary to my clipboard. Use built-in browser APIs only.
I18N Add a language toggle
Add a toggle in the top-right that switches the interface between English and Bahasa Melayu. Translate all labels and buttons cleanly.
WHEN THINGS GO SIDEWAYS

Iteration got messy?

The most common rough patches when you're changing a working app — and the one-line fix for each.

🔄
AI broke my working app
▸

Don't panic. Say: Revert the last change — go back to the previous working version.

Still wrong? Scroll up in your chat to the last good code block and copy that one instead. Your old code is still there in the chat history.

🔗
My Netlify URL changes every deploy
▸

Each fresh drop creates a new "site". To fix it once:

Create a free Netlify account → on your most recent deploy, click Site settings → Change site name → pick something memorable like my-tracker-app. Now redeploying keeps the URL.

📦
App is getting too complex / slow
▸

Single-file HTML hits a ceiling around 1,000–1,500 lines of code. Past that, your AI may struggle to hold all the pieces together.

Two options: (a) trim features — keep only what you really use, or (b) move to a real repo and database — see Tracks 2–4.

🎨
New feature looks off — design drifted
▸

Design drift is real. Two prompts that fix it:

Restyle the new feature to match the existing design language.

Polish the whole UI — keep all functionality identical but make the visual design cohesive and modern.

⚡
Tempted to add multiple features at once
▸

Resist the urge. Asking for 3 features in one prompt has a much higher break rate.

Add one. Test it. Save the working version. Then add the next.
