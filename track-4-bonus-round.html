Open Codespaces — watch Copilot read your AGENTS.md

Codespaces is a full VS Code editor that runs in your browser, attached to your GitHub repo. No install, no setup — you get a real editor, terminal, and AI assistant on any device. And the moment you open Copilot, the AGENTS.md you wrote in Track 3 starts paying you back.

WHY THIS PATH

Until now you've been pasting code in and out of a chat. Codespaces gives you a real editor with an AI that can actually see your whole project — including your AGENTS.md. The conventions you wrote down get applied automatically.

PREREQ

The GitHub repo from Track 2, with the AGENTS.md from Track 3 committed at its root. Codespaces uses GitHub's free hours (120 hr/month).

A.01
Open Codespaces from your repo

Go to your repo on github.com. Click the green Code button → Codespaces tab → Create codespace on main.

Wait ~60 seconds. A full VS Code editor loads in your browser, with your whole repo already cloned in.

A.02
Open your index.html

In the left sidebar (the file explorer), click index.html. The full code opens in the editor — with syntax highlighting, autocomplete, and Git built in.

You should also see AGENTS.md sitting right there in the file list. That's the file that's about to do the heavy lifting.

A.03
Preview your app live

Open the terminal (View menu → Terminal, or Ctrl+`). Run:

$ python3 -m http.server 8000

Codespaces will pop up an "Open in Browser" notification. Click it. Your app loads — now you can edit, save (Ctrl+S), refresh, and see changes instantly, no redeploying.

A.04
Install GitHub Copilot

Left sidebar → Extensions icon → search "GitHub Copilot" → Install. Also install "GitHub Copilot Chat" — the conversational sibling. (Free for students and verified open-source maintainers; otherwise free trial then paid.)

Sign in when prompted. You should now see Copilot suggestions appear inline as you type, and a chat icon appear in the sidebar.

A.05
Watch Copilot read your AGENTS.md

This is the payoff moment.

Open Copilot Chat (sidebar icon or Ctrl+Alt+I). Don't paste any context. Just ask something vague like:

FRESH_PROMPT Add a contact form to my site.

Copilot replies — and references your conventions. The colours you picked. The "no external libraries" rule. The structure of your single HTML file. You didn't say any of that in this chat.

It read AGENTS.md from your repo before answering. The brief you wrote in Track 3 is now doing its job, hands-free, for every future session.

Try one more: ask "what does this project do?" — you'll get back a tight summary that matches your overview almost word-for-word. That's the file working.

A.06
Commit, push — and watch it go live

Made changes you want to keep? Left sidebar → Source Control icon (the branchy symbol) → type a commit message → Commit → Sync.

Your code is now pushed to GitHub. The Netlify connection you set up in Track 2 picks it up automatically — a new deploy fires within ~30 seconds.

Open your Netlify dashboard → find your site → Deploys tab. You'll see the new deploy triggered. When it shows "Published", your live URL is updated.

The full loop: edit in Codespaces → commit → push → Netlify rebuilds → live. No manual upload, ever again.

OUTCOME: A real developer setup — editor, terminal, AI assistant, version control — all in your browser. Every commit goes live automatically. Your AGENTS.md briefs every Copilot session, no setup needed.
// PATH_B · 30 MIN

Add a real database with Supabase

Until now, your app forgot everything when the browser closed (or kept it only on one device via localStorage). Supabase gives you a free hosted database — your app can now remember things across devices, across users, across time.

WHY THIS PATH

Single-file HTML hits a real ceiling when you need persistence beyond one browser. Supabase lifts that ceiling without changing your stack.

PREREQ

A working app from Track 1 with some kind of data (entries, todos, scores, anything).

B.01
Create a Supabase project

Sign up at supabase.com (free tier is generous — no credit card needed). Click New project. Pick a name, set a database password (save it somewhere safe), choose the Singapore region for low latency.

Wait ~2 minutes for it to provision.

B.02
Pick a feature that needs a database

The right feature is one your app couldn't do with localStorage alone. Examples:

• Shared notes wall — everyone using the app sees everyone's notes
• Cross-device sync — log your savings from phone, see it on laptop
• Public leaderboard — quiz scores everyone can see
• Comment thread — visitors leave feedback on your tool

B.03
Get your project keys

In Supabase dashboard → Project Settings → API. You need two things:

• Project URL — looks like https://abcdefg.supabase.co
• publishable key — starts with sb_publishable_... (older projects show it as the anon key — a JWT starting with eyJh...). Both work the same way.

Copy both. You're about to put them in your HTML file so the app can talk to your database.

⚠ TWO KEYS, ONE FOOTGUN: Supabase gives you two keys. One is meant to be public, the other is the one to fear.

Publishable / anon key — safe to ship. This is the one you paste into your HTML. It's designed to be public — any user can view-source your site and see it. Supabase calls it "publishable" for exactly that reason.

What keeps it safe: Row-Level Security policies. The key can only do what your policies allow. Without policies, the key is harmless because every request gets denied. With policies, the key gets exactly the access you granted — nothing more.

Secret key (sb_secret_... or service_role) — never in client code. This one bypasses every policy. It belongs in server-side code, environment variables, or an Edge Function. Never paste it into an HTML file, never commit it to GitHub, never share it in chat.

The real rule of thumb: if a key starts with publishable_ or is labelled "anon", ship it. If it starts with secret_ or is labelled "service_role", treat it like a password.
B.04
Ask your AI to wire it up

Go back to Claude/ChatGPT with your current app. Send this prompt (filling in your actual values):

PROMPT_TEMPLATE Add Supabase to my existing app. Create a table called entries with columns: id (uuid, primary key), created_at (timestamp), content (text). Replace localStorage with Supabase reads and writes. Use the Supabase JS client loaded from a CDN — keep this as a single HTML file. My Supabase URL is YOUR_URL_HERE and my anon key is YOUR_ANON_KEY_HERE. Also write me the SQL to create the table so I can run it in the Supabase SQL editor.
B.05
Create the table in Supabase

Your AI will give you SQL like CREATE TABLE entries (...). Go to Supabase dashboard → SQL Editor → paste the SQL → Run.

Then go to Authentication → Policies → for the entries table, enable a basic policy allowing reads/writes (Supabase will guide you with templates).

B.06
Deploy and test

Copy the updated HTML, redeploy (same flow as Track 1, or commit it to your Track 2 repo for auto-deploy). Open the URL on two devices. Add an entry from one — refresh the other. Magic.

OUTCOME: Your app now has a real backend. Data persists forever, syncs across devices, is shareable with the world.
// PATH_C · 45 MIN

Install Claude Code CLI + Superpowers

Claude Code is a terminal-based coding agent that operates on your actual codebase — reading files, running commands, making edits. Superpowers is a plugin that adds structured methodology (brainstorming, planning, TDD, systematic debugging). Together, the most capable AI-assisted dev setup currently available.

WHY THIS PATH

This is what your most-capable engineer friends are using on weekends. The ceiling is genuinely high. Steep curve, but everything before this was preparing you.

PREREQ

A laptop (not a phone). A terminal you're willing to look at. Node.js installed.

C.01
Open a terminal

🪟 Windows: Press Win+R → type powershell → Enter. Or search "Terminal" in Start menu.

🍎 Mac: Press Cmd+Space → type Terminal → Enter.

You'll see a window with a blinking cursor. This is where you'll type commands.

C.02
Install Node.js (if you don't have it)

Check first:

$ node --version # If you see v18.x or higher, skip to C.03.

If not installed, go to nodejs.org and install the LTS version (works on Windows + Mac). Restart your terminal after.

C.03
Install Claude Code

In your terminal:

$ npm install -g @anthropic-ai/claude-code

Wait for it to finish (~1 minute). Verify:

$ claude --version
C.04
Navigate to your repo & launch

Use the same repo you set up in Track 2 — this way any changes Claude Code makes will auto-deploy to your live Netlify site.

First, clone your repo to your laptop if you haven't already:

$ git clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git $ cd YOUR_REPO_NAME

Then launch Claude Code inside that folder:

$ claude

First-run will prompt you to log in via your browser. Sign in with your Anthropic account (Claude Pro/Max subscription, or API credits).

The Netlify connection: When Claude Code edits your files and you commit + push (Claude Code can do this for you — just ask it to), Netlify detects the change and rebuilds your live site automatically within ~30 seconds. You can watch it happen in your Netlify dashboard under Deploys.

C.05
Try a first task

Inside Claude Code's prompt, try something concrete to feel how it differs:

PROMPT Read my index.html. Add a dark mode toggle in the top right corner. Use CSS variables. Remember the user's choice with localStorage.

Claude reads your file, makes edits, and shows you a diff before writing. Approve the changes. This is already different from what you've been doing.

C.06
Install Superpowers

Still inside Claude Code, run:

claude> /plugin install superpowers@claude-plugins-official

This adds 14 structured skills to Claude Code — brainstorming, planning, test-driven development, systematic debugging, code review. The skills activate automatically based on what you're doing.

Quit and restart Claude Code. You'll see a startup message confirming Superpowers is loaded.

C.07
Try the superpower: /superpowers:brainstorm

Instead of immediately asking Claude to build something, let it brainstorm with you:

claude> /superpowers:brainstorm

Describe a vague idea — something bigger than what you've built so far. For example: "I want to track my freelance work and invoice clients."

Claude won't write code. It'll ask Socratic questions, propose alternatives, narrow the spec. Once you approve the design, try /superpowers:write-plan → it produces a detailed implementation plan. Then /superpowers:execute-plan to build it, with built-in code review at every step.

OUTCOME: You've just experienced the shift from "AI as autocomplete" to "AI as a structured collaborator." This is what AI-assisted engineering looks like at the senior level.
